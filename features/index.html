The full stack

Every feature, explained.

three.ws is one stack — a WebGL viewer, an LLM-powered agent runtime, on-chain identity, and an embed layer — built to make a 3D AI agent of yourself that lives anywhere on the internet and is owned by you.

01 — 3D Viewer

A real 3D engine, in the browser.

Built on three.js r176 and WebGL 2.0. Drop a GLB or glTF file in and it renders instantly — no plugin, no upload, no server round-trip. The same viewer powers the platform, the embed widgets, and the validator.

🖼️

WebGL 2.0 rendering

Hardware-accelerated rendering via three.js r176. PBR materials, HDR environment maps, real-time lighting and shadows — running on the GPU in any modern browser.

📦

glTF 2.0 + GLB

Full glTF 2.0 / GLB pipeline with Draco geometry compression, KTX2 (Basis Universal) texture compression, and Meshopt mesh optimization for fast loads at scale.

✔️

Khronos validation

Every uploaded file is validated against the Khronos glTF spec with line-level error reporting, surfaced in the on-platform validator at /validation.

🎬

Animations & morph targets

Skinned mesh animations with bone retargeting, morph targets for facial expressions, and embedded camera tracks — all driven from the agent runtime by name.

🎛️

OrbitControls + auto-rotate

Pan, zoom, and rotate out of the box. Configurable auto-rotation for showcase modes. Touch-friendly on mobile, with sensible defaults for embedded contexts.

🎚️

Live parameter tweaking

Real-time controls for lights, exposure, and morph weights via a dat.GUI panel — useful for tuning an avatar before publish or debugging an embedded agent.

02 — Agent Runtime

An LLM brain wired into a body.

Claude is the default model, with a structured tool-loop that lets the agent decide on actions, call tools, observe results, and respond. Every output is routed through the avatar — speech, gestures, expression, and gaze.

🧠

Claude tool-loop

Anthropic Claude as the default brain, with a structured tool-loop. Up to 8 tool iterations per turn before returning final output to the user.

🛠️

Built-in tools

wave, lookAt, play_clip, setExpression, speak, and remember — the primitives every embodied agent gets for free.

🧩

Composable skills

Install skills from IPFS, Arweave, or HTTP. Each skill is a self-contained bundle: description, tool definitions, and async handlers. No redeploy required.

🎭

Weighted emotion blending

Celebration, concern, curiosity, empathy, and patience are blended continuously based on protocol events — not flipped through a finite-state machine.

🎙️

Voice (STT + TTS)

Web Speech API for STT/TTS out of the box. ElevenLabs for production-quality voice. LiveKit for bidirectional realtime audio rooms with low-latency duplex.

⚡

Solana Blinks

Agents can generate and execute Solana Blink actions from shareable URLs — turning a chat reply into a signed on-chain transaction the user can confirm in one click.

03 — On-Chain Identity

Your agent is an economic actor, not a PFP.

Every agent gets a stable on-chain ID, a delegated signer wallet, a signed action history, and a reputation score. Built on ERC-8004 for EVM, with first-class Solana support via SIWS and Metaplex Core.

⛓️

ERC-8004 contracts

IdentityRegistry, ReputationRegistry, and ValidationRegistry — deployable on any EVM chain. Open-source, auditable, and already running on Base.

🏷️

ERC-721 token per agent

Each agent is an ERC-721 with a stable agentId, an owner wallet, a delegated signer, and an IPFS-pinned manifest containing body, persona, and skills.

🔐

EIP-712 delegated signer

The agent transacts autonomously within a permission scope you define. Revocable, scoped, and EIP-712 signed — composable with EIP-7710 delegation chains.

📜

Signed action log

Every speak, remember, skill-done, and validate event is cryptographically signed and recorded — on-chain optionally, in the database always.

📊

Reputation score

Built from real signed actions, not self-reported. Publicly readable through ReputationRegistry. Stakes, royalties, and discovery weight all follow the score.

🌍

Solana support

Sign-In With Solana (SIWS), Solana wallet linking, and an optional Metaplex Core NFT path for agents that want to live on Solana instead of EVM.

04 — Embedding & Distribution

One line of HTML. Anywhere on the web.

The <agent-3d> custom element is a zero-framework drop-in. Five ready-made widget variants cover the common cases. Versioned CDN bundles and rich social previews are wired in by default. 

<!-- Drop into any page -->
<script src="https://three.ws/agent-3d/latest/agent-3d.js"></script>
<agent-3d src="https://three.ws/agent/your-agent"></agent-3d>
🧱

<agent-3d> web component

A native custom element. No React, no Vue, no build step. Works in any page that loads a script tag — including static sites, Notion embeds, and CMS templates.

🔃

Turntable widget

Auto-rotating showcase view. Best for marketplaces, profile pages, and product hero sections where you want the model itself to do the work.

🎬

Animation gallery

Browsable list of every animation clip in the GLB, with one-click playback. Useful for character portfolios and rigging review.

💬

Talking agent

Full chat surface with voice input/output and lip sync wired to the LLM brain. The default if you want visitors to actually converse with the agent.

🆔

ERC-8004 passport card

A compact card showing the agent ID, owner, reputation score, and signer status — sourced live from the on-chain registry. Drop-in proof of identity.

📍

Hotspot tour

Annotate points on the model with text, links, or skill calls. Click a hotspot to fly the camera and trigger an action. Built for product walkthroughs.

🎛️

Widget Studio

Point-and-click embed code generator at /studio. Pick a widget, configure it, and copy the snippet — published widgets get their own public URL at /w/[id].

🔗

oEmbed + Open Graph

Rich social previews on X, Discord, Slack, and Telegram. oEmbed endpoint and per-agent OG images render on demand so every shared link looks right.

📡

Versioned CDN

Pinned bundles at /agent-3d/x.y.z/agent-3d.js with year-long immutable cache. Use latest for evergreen embeds, an exact semver for production.

05 — Backend & Integrations

Standards-first. Real protocols, not custom ones.

A full OAuth 2.1 server, an MCP endpoint that lets external AI systems call agent tools over JSON-RPC, and turnkey integrations with the avatar and wallet vendors developers already use.

🔑

OAuth 2.1 server

RFC 6749 + PKCE, RFC 7591 dynamic client registration, RFC 7009 revocation, RFC 7662 introspection, RFC 8414 discovery. Full spec, no shortcuts.

🧪

Developer API keys

Scoped API keys with expiry, issued through the dashboard. Use them for server-side calls when an OAuth flow would be overkill.

🔌

MCP endpoint

Model Context Protocol over HTTP with JSON-RPC 2.0. External AI systems — Claude Desktop, Cursor, custom agents — can call your agent's tools as remote skills.

👤

Ready Player Me

Import a Ready Player Me avatar URL directly into the wizard. The model is validated, optimized, and pinned to your account in one step.

📸

Avaturn (photo to avatar)

Snap a photo, get a rigged 3D avatar back. Avaturn sessions are brokered server-side so credentials never touch the browser.

💳

Privy embedded wallets

Email login that ships with a non-custodial wallet — for users who want to mint an agent without already owning crypto. Linked to ERC-8004 identity.

📚

OpenAPI 3.1 spec

Generated live at /openapi.json. Wire any code-gen tool, Postman, or LLM-assisted client to the full surface area without hand-rolling types.

📅

DCA & subscriptions

On-chain DCA strategies and recurring subscription payments, scheduled by cron. Designed at /strategy-lab, executed without your wallet being online.

📨

Well-known discovery

Standard discovery endpoints under /.well-known — agent-card, x402 payments, OAuth metadata, attestation schemas — for protocol-level interop.
