How It Works

01

Register

Mint an ERC-721 token on any EVM chain. The tokenURI points to your agent's JSON registration file — name, description, endpoints, and trust preferences.

02

Discover

Other agents and clients query the Identity Registry to find your agent. Service endpoints expose A2A, MCP, web interfaces, ENS names, and wallet addresses.

03

Trust

Reputation feedback, validation registries, and cryptographic proofs build trust proportional to value at risk — from ordering pizza to medical diagnosis.

Agent Registration File

Every 3D avatar is a JSON file — the same format ERC-8004 uses for agent identity. Your avatar's metadata becomes its on-chain passport.

agent-registration.json 
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "3D Agent",
  "description": "AI-powered 3D model viewer and validation agent",
  "image": "https://3dagent.vercel.app/og-image.png",
  "active": true,
  "services": [
    { "name": "web",  "endpoint": "https://3dagent.vercel.app/" },
    { "name": "A2A",  "endpoint": "https://3dagent.vercel.app/.well-known/agent-card.json" },
    { "name": "MCP",  "endpoint": "https://3dagent.vercel.app/.well-known/openapi.yaml" }
  ],
  "supportedTrust": ["reputation"]
}

Three Registries

🪪

Identity Registry

ERC-721 based. Every agent gets a unique agentId and a URI that resolves to its registration file. Transferable, composable, browsable with any NFT app.

  • register(agentURI)
  • setAgentURI()
  • setMetadata()
⭐

Reputation Registry

Any client can leave signed feedback — quality scores, uptime, response time. On-chain composability meets off-chain aggregation.

  • giveFeedback(agentId, value)
  • getSummary(agentId)
  • revokeFeedback()
🔐

Validation Registry

Request independent verification: stake-secured re-execution, zkML proofs, TEE attestations. Trust proportional to value at risk.

  • validationRequest()
  • validationResponse()
  • getValidationStatus()

3D Avatars Meet Web3 Identity

GLB → On-Chain

Your 3D avatar's GLB file becomes the "image" field in the registration JSON. Render it in any viewer, verify it on-chain.

Validation Built-In

glTF validation results feed the Reputation Registry — model quality scores become on-chain reputation signals.

Agent Discovery

Any A2A or MCP client can discover your 3D Agent via the Identity Registry and interact through standard endpoints.

Portable & Composable

Your agent identity is an NFT — transfer it, delegate it, list it. Works across any EVM chain.
