# Prompt 7: Backend - Verify and Record Purchase

**Status:** Not Started

## Objective
Create a backend endpoint to verify a completed Solana transaction and record the skill purchase in the database.

## Explanation
After a user approves a transaction through the Solana Pay QR code, we need a way to confirm that the payment was successful. Solana Pay includes a mechanism for this. The transaction can be found on-chain and, once verified, the purchase can be recorded in our `user_skill_purchases` table. This endpoint will be responsible for this verification and recording process.

## Instructions
1.  **Create a verification endpoint:**
    - Create a new API route, e.g., `/api/skills/verify-purchase`.
    - It should accept a `POST` request with the `transaction_signature`.

2.  **Verify the transaction:**
    - In the endpoint, use the Solana Web3 library (`@solana/web3.js`) to fetch the transaction details using the provided signature.
    - **Crucially, verify all aspects of the transaction:**
        - That the transaction was successful.
        - That the correct amount of the correct SPL token was transferred.
        - That the token was transferred to the correct recipient (the agent creator).
        - That the `reference` public key in the transaction matches the one you generated in the purchase initiation step.

3.  **Record the purchase:**
    - If the transaction is valid, insert a new record into the `user_skill_purchases` table.
    - The record should include the `user_id`, `agent_id`, `skill_name`, price, currency, and the transaction signature.

4.  **Polling from the frontend:**
    - The frontend, after displaying the Solana Pay QR code, will need to poll this verification endpoint to know when the purchase is complete.
    - The frontend will send the transaction signature (which it can get from the Solana Pay `findTransactionSignature` method) to the backend.
    - When the backend confirms the purchase, the frontend can close the modal and update the UI to show the skill as "Owned."

## Code Example (`api/skills/verify-purchase.js`)

```javascript
// api/skills/verify-purchase.js
import { Connection, PublicKey } from '@solana/web3.js';

export default async function handler(req, res) {
  // ... authentication and request validation ...

  const { transactionSignature, agentId, skillName } = req.body;
  const userId = await getUserIdFromSession(req);

  // 1. Fetch transaction from Solana blockchain
  const connection = new Connection(process.env.SOLANA_RPC_URL);
  const transaction = await connection.getTransaction(transactionSignature);

  // 2. Verify transaction details (this is a simplified example)
  //    In a real app, you would need to parse the transaction instructions
  //    to confirm the amount, mint, and recipient are correct.
  if (transaction && !transaction.meta.err) {
    // 3. Record the purchase
    const price = await getSkillPrice(agentId, skillName); // You need this function
    await db.query(
      `INSERT INTO user_skill_purchases (user_id, agent_id, skill_name, purchase_price, purchase_currency, transaction_signature)
       VALUES ($1, $2, $3, $4, $5, $6)`,
      [userId, agentId, skillName, price.amount, price.currency_mint, transactionSignature]
    );

    return res.status(200).json({ success: true, message: 'Purchase successful' });
  } else {
    return res.status(400).json({ error: 'Transaction verification failed' });
  }
}
```
