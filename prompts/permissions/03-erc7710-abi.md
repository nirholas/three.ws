# Task 03 — ERC-7710 `DelegationManager` ABIs and chain meta

## Why

Every other task that touches on-chain delegation (grant UI, redeem hook, revoke endpoint, indexer, verify endpoint) needs a single source of truth for the `DelegationManager` address per chain and the ABI fragments used. Mirrors the pattern in [src/erc8004/abi.js](../../src/erc8004/abi.js).

## Read first

- [src/erc8004/abi.js](../../src/erc8004/abi.js) — use this as the template for structure, comment style, and human-readable ethers v6 strings
- [src/erc8004/chain-meta.js](../../src/erc8004/chain-meta.js) — see how chain metadata is organized; reuse constants where possible
- MetaMask Delegation Toolkit docs (<https://docs.metamask.io/delegation-toolkit/>) — authoritative source for contract addresses per chain and ABI. **Do not guess addresses.** If the docs don't list a chain, do not include it.
- [00-README.md](./00-README.md) — canonical exports from `src/erc7710/abi.js`

## Build this

Create `src/erc7710/abi.js` with:

1. `DELEGATION_MANAGER_DEPLOYMENTS` — object keyed by numeric chainId, values are EIP-55 checksummed addresses. Include at minimum: Base Sepolia (84532), Sepolia (11155111), Base mainnet (8453), Ethereum mainnet (1). Source addresses from the MetaMask docs; comment each with the date you fetched and the docs URL section.
2. `DELEGATION_MANAGER_ABI` — ethers v6 human-readable string array. Include at least: `redeemDelegations`, `disableDelegation`, `isDelegationDisabled`, `getDelegationHash`, and the events `DelegationRedeemed`, `DelegationDisabled`. Verify signatures against the MetaMask docs/Solidity source.
3. `CAVEAT_ENFORCERS` — object mapping enforcer name (`AllowedTargetsEnforcer`, `ERC20LimitEnforcer`, `TimestampEnforcer`, `NonceEnforcer`, etc.) → per-chain address map. Same EIP-55 checksumming.
4. `EIP712_DOMAIN({ chainId, verifyingContract })` helper returning the domain separator fields used for delegation signing.
5. `encodeCaveats(caveats)` — pure function that ABI-encodes an array of caveats (`{ enforcer, terms, args }`) into the on-chain format. Use ethers `AbiCoder`.
6. Optional: `DELEGATION_TYPES` — the EIP-712 `types` object for signing (keys: `Delegation`, `Caveat`).

File conventions:

- ESM only. Match prettier style (tabs, single quotes, 100 cols).
- No runtime dependencies other than `ethers` (already in the project).
- Export each symbol as a named export.
- Every exported constant has a 1-line comment above it documenting its purpose + source.

Add a second file `src/erc7710/README.md` (15–40 lines) listing: what this module is, what's NOT in it (no signing, no RPC — just constants + pure encoders), and how `src/permissions/toolkit.js` consumes it.

Finally add a self-check script `scripts/check-erc7710-addresses.js` that:

- Imports `DELEGATION_MANAGER_DEPLOYMENTS`.
- For each chain, calls `eth_getCode` via the public RPC (pull URL from `src/erc8004/chain-meta.js` if present; else inline Base/Sepolia public RPCs).
- Fails non-zero if any address returns `0x` (i.e. not a contract).
- Run it yourself; paste the output in the reporting block.

## Don't do this

- Do not invent contract addresses. If MetaMask docs don't list Base mainnet yet, omit it with a comment.
- Do not pull in the full `@metamask/delegation-toolkit` in this file — that goes in `src/permissions/toolkit.js` (task 04). This file is constants + pure encoders only.
- Do not use ABI JSON. Stick to human-readable strings for consistency with the rest of the repo.
- Do not mix checksummed and lower-cased addresses in the same map.
