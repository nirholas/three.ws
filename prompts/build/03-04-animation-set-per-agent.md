# 03-04 — Animation set per agent

## Why it matters

The Empathy Layer (idle, wave, nod, shake, think) is the difference between a static model and an embodied agent. Today the Mixamo bundle is global and ad-hoc. Each agent needs a *chosen, persisted* animation set so the avatar reads the same across the agent page, embeds, and host integrations (Layers 4–5).

## Context

- Animation manager: [src/animation-manager.js](../../src/animation-manager.js).
- Fetch script: `npm run fetch-animations` (Mixamo bundle in [public/animations/](../../public/animations/)).
- Manifest spec: [specs/AGENT_MANIFEST.md](../../specs/AGENT_MANIFEST.md).
- Per-agent record: `agents` table in [api/_lib/schema.sql](../../api/_lib/schema.sql).

## What to build

### Schema

```sql
alter table agents add column if not exists animation_set jsonb not null default '{}'::jsonb;
```

Shape: `{ idle: "breathing", wave: "wave-hello", nod: "nod-yes", shake: "shake-no", think: "thinking" }`. Keys are the canonical Empathy Layer slots (defined in the spec); values are animation clip names that exist in the shared bundle.

### Endpoint — `PUT /api/agents/:id/animations`

- Owner-authed. Body validates against a whitelist of clip names served by `GET /api/animations/catalog` (new endpoint that lists clips in the bundle).
- Reject unknown clip names with 400. Missing slots inherit the defaults from the spec.
- Bump `agents.updated_at`.

### Catalog endpoint — `GET /api/animations/catalog`

- Public. Returns `[{ name, duration_ms, category, preview_url }]` for every clip in the bundle. Cacheable for 1h.
- Source of truth: a static JSON committed alongside the bundle, regenerated by `fetch-animations`.

### Dashboard UI — animation picker

- New "Animations" tab on the agent detail page in [public/dashboard/](../../public/dashboard/).
- Each Empathy slot is a dropdown of compatible clips (filtered by `category`).
- Inline preview: picking a clip plays it on an inline `<model-viewer>`.
- Save button PUTs the set; optimistic update with rollback on error.

### Runtime consumption

[src/agent-avatar.js](../../src/agent-avatar.js) reads `manifest.animation_set` and wires each Empathy bus event to the chosen clip. If a slot is missing, fall back to the spec default. No behavioral change for agents that haven't picked yet.

## Out of scope

- Authoring new animations (upload / blending). Bundle is read-only.
- Per-event timing tuning or blend weights.
- Animation marketplaces or sharing across users.

## Acceptance

1. A new agent uses defaults; the avatar idles and waves as before.
2. Swap the "nod" slot to a different clip → save → reload the agent page → the new clip plays on nod.
3. Swap to a clip that doesn't exist via curl → 400, record unchanged.
4. `GET /api/animations/catalog` lists every clip in [public/animations/](../../public/animations/) and the list survives a hard refresh (cached).
5. `node --check` passes on modified files.
