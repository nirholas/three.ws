Sign in with three.ws

This page imitates a third-party tenant site. Clicking the button below opens a popup to the three.ws consent screen and — on approval — receives a 24-hour token plus the user's avatar URL via postMessage. The token's audience is bound to this origin so it cannot be replayed against another *.three.ws site.

3. Server-side verification (snippet)

Tenants verify tokens before trusting them. Audience is enforced to prevent cross-tenant replay. 

const audience = location.origin;
const res = await fetch(
  `https://three.ws/api/auth/persona/verify?token=${encodeURIComponent(token)}` +
  `&audience=${encodeURIComponent(audience)}`,
);
const claims = await res.json();
if (!res.ok) throw new Error(claims.error_description);
// claims.sub        → three.ws user id
// claims.avatar     → { id, url, thumbnail_url, name }
// claims.scope      → "persona:read avatar:read"
