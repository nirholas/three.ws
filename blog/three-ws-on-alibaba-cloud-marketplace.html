three.ws Launches on Alibaba Cloud Marketplace, Bringing Native x402 Agent Payments to Hyperscale

May 17, 2026 · three.ws

three.ws is now live on the Alibaba Cloud International Marketplace. The listing makes <agent-3d> — a single HTML tag that deploys a fully on-chain 3D AI agent — available to every Alibaba Cloud customer worldwide.

This is the first deployment of on-chain agents with native HTTP 402 / x402 payment capability inside the procurement flow of a top-four cloud provider.

What three.ws ships

three.ws is an open-source stack for autonomous 3D AI agents that run natively in the browser. Every <agent-3d> tag deploys an agent with:

  • A Solana NFT identity. Each agent is minted as a transferable on-chain asset.
  • An ERC-8004 wallet. The emerging cross-chain agent-identity standard, baked in.
  • A browser-native 3D body. Rendered in WebGL — no plugins, no native install, no client runtime.
  • A swappable LLM brain. Claude, GPT, and open-weights models are live today. Qwen and ModelScope land through the Alibaba Cloud integration.
  • Native x402 / HTTP 402 payments. Agents pay for what they consume and earn for what they offer, settling in USDC on Solana.

A developer integrates an agent into any web page with a single line of HTML. That tag is a Solana-native economic actor — it can hold value, sign transactions, charge for skills, pay for inference, and route revenue back to its owner with no backend, no API keys, no Stripe account, and no checkout page.

The Alibaba Cloud listing

Alibaba Cloud is the #1 cloud provider in Asia and one of the top four hyperscalers globally. Its International Marketplace is the procurement gateway used by enterprise customers across China, Southeast Asia, the Middle East, and Latin America — regions where AWS and Google Cloud run second.

A marketplace listing is the actual buying flow that enterprises use to acquire software through their existing cloud account, with consolidated billing, compliance, and support. three.ws now sits inside that flow as a directly procurable product.

Why this matters for the agent economy

HTTP 402 — "Payment Required" — has been reserved in the HTTP specification since 1997 for machine-to-machine payments. The x402 protocol activated it on-chain. three.ws makes x402 the default payment layer for embedded 3D agents.

Three structural outcomes follow from the Alibaba Cloud listing:

  1. Distribution at hyperscale. The Marketplace exposes three.ws to millions of paying enterprise developers across 200+ countries.
  2. Native on-chain settlement. Every agent action — minting, skill purchases, inference, accessory acquisition, coin launches — routes through Solana via three.ws rails. No off-chain accounting, no payment processors.
  3. Portable agent identity. ERC-8004 gives each agent a cross-chain wallet identity, removing the need for proprietary account systems and making agents interoperable across the wider on-chain agent ecosystem.
